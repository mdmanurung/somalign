# Plan 08 — Anchor Exclusion-Restriction Test (Sargan–Hansen Analog)

**Feature:** `somalign_exclusion_test()` — a permutation test that the anchor
displacement matrix D carries no structured signal orthogonal to the estimated
batch subspace V.

---

## 1. Summary

In IV/negative-control language the batch correction is valid only if anchors
satisfy the exclusion restriction: the component of D = anchor_old_scaled −
anchor_new_scaled that is *orthogonal* to the batch subspace V should be
unstructured noise, not systematic biology-×-batch signal. This test operationalises
that assumption.

The statistic is the leading singular value of the orthogonal residual
`R = D (I − V Vᵀ)` (n_anchors × p matrix). Under H₀ (pure noise in the
orthogonal complement) this statistic should not exceed what a permutation null
produces. The null is generated by shuffling rows of `anchor_old_scaled` (breaking
within-pair correspondence while preserving marginal distributions), recomputing
D_perm, projecting onto the same V, and recording the leading SV each time. A
one-sided p-value is `mean(sv_null >= sv_obs)`.

A passing test (p > 0.05) supports the IV assumption and strengthens the scientific
credibility of the subspace correction. Failure (p < 0.05) warns that the subspace
correction may be removing biology, and suggests increasing anchor diversity or
switching to `correction = "cost_bonus"`.

---

## 2. Public API

### Function signature

```r
somalign_exclusion_test(fit, n_perm = 999L, seed = 1L)
```

**Chosen form:** accepts a fitted `somalign_anchored_fit` object (not raw matrices).
This is preferred over the raw-anchor form because:

1. V is already computed and stored in `fit$anchors$batch_subspace$V`; recomputing
   it from raw inputs with a user-supplied `variance_threshold` would produce a
   different V than the one used for the actual correction, making the test
   incoherent.
2. The raw-anchor form would duplicate validation logic already in
   `.somalign_validate_anchors()`.
3. It follows the precedent of `somalign_diagnostics(fit)`.

The function requires `fit$anchors$correction %in% c("subspace", "both")` and
`fit$anchors$displacements` (see §3 below). It errors informatively if called on a
`"cost_bonus"` fit (no subspace was estimated) or an unanchored fit.

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `fit` | `somalign_anchored_fit` | Output of `somalign_fit_anchored(..., correction = "subspace"/"both")` |
| `n_perm` | positive integer | Number of permutation replicates (default 999) |
| `seed` | integer or NULL | RNG seed for reproducibility (default 1L); NULL = no seeding |

### Return value

A named list of class `"somalign_exclusion_test"`:

```r
list(
  sv_observed      = <scalar>          # leading SV of R = D (I - V Vᵀ)
  sv_null          = <numeric n_perm>  # leading SVs under permutation null
  p_value          = <scalar in [0,1]> # mean(sv_null >= sv_observed)
  null_quantiles   = <named numeric>   # 2.5 / 50 / 97.5 percentiles of sv_null
  relative_stat    = <scalar>          # sv_observed / sv_null_median (effect size)
  rank_used        = <integer>         # batch subspace rank (ncol(V))
  n_anchors        = <integer>         # n rows of D
  n_features       = <integer>         # p
  verdict          = <character>       # "pass" / "warn" / "fail"
  feature_residual_norm = <named numeric p>  # per-feature ||R[,j]||_2
)
```

**Verdict thresholds:** "pass" = p > 0.1, "warn" = p in [0.05, 0.1], "fail" = p < 0.05.
`feature_residual_norm` identifies which markers drive any violation (useful diagnostics
for the fail/warn case).

A `print.somalign_exclusion_test()` S3 method prints a one-line summary:
`"Exclusion-restriction test: sv_obs = X.XX, p = 0.NNN [pass/warn/fail]"`.

## 3. Prerequisite Data-Structure Change

The test needs the scaled displacement matrix D (n_anchors × p). Currently D is
computed as `anchor_old_scaled - anchor_new_scaled` inside `.somalign_batch_subspace()`
(anchored.R line 249) and then discarded — it is never stored in `fit$anchors`.

### Option A — store D_scaled at dispatch time (preferred, shared with Idea 7)

In `.somalign_anchored_dispatch()` (anchored.R lines 191–245), extend the `anchors`
list passed to `.somalign_finish_fit()` to include `displacements`:

**Exact edit — anchored.R, inside `.somalign_anchored_dispatch()`, lines 230–242:**

```r
# BEFORE (lines 234–242):
  fit <- .somalign_finish_fit(
    query, reference, transport, min_match_fraction, confidence_threshold,
    correction_min_mass, chunk_size, epsilon, rho_query, rho_ref,
    shift_transform = shift_fn,
    anchors = list(
      n_anchors         = nrow(anchors_scaled$anchor_old_scaled),
      rho_anchor        = rho_anchor,
      correction        = correction,
      nodes_covered     = cb$nodes_covered,
      coverage_fraction = cb$coverage_fraction,
      batch_subspace    = batch_sub
    )
  )

# AFTER:
  D_scaled <- anchors_scaled$anchor_old_scaled - anchors_scaled$anchor_new_scaled
  fit <- .somalign_finish_fit(
    query, reference, transport, min_match_fraction, confidence_threshold,
    correction_min_mass, chunk_size, epsilon, rho_query, rho_ref,
    shift_transform = shift_fn,
    anchors = list(
      n_anchors         = nrow(anchors_scaled$anchor_old_scaled),
      rho_anchor        = rho_anchor,
      correction        = correction,
      nodes_covered     = cb$nodes_covered,
      coverage_fraction = cb$coverage_fraction,
      batch_subspace    = batch_sub,
      displacements     = D_scaled          # NEW: n_anchors × p
    )
  )
```

Storage overhead: for typical CyTOF data n_anchors < 200, p < 50, so D_scaled is
at most 200 × 50 × 8 bytes = 80 KB — negligible.

**This is the only change to existing code.** It is backward-compatible (adding a
new list element does not break existing downstream code).

### Option B — recompute on demand

`somalign_exclusion_test()` could call `.somalign_validate_anchors()` itself and
require the user to pass `anchor_old` and `anchor_new` again. Rejected: the
raw-anchor form is less ergonomic, requires the user to track their inputs, and as
noted in §2 produces a different V if `variance_threshold` differs. Option A is
strictly better.

## 4. Internal Helpers

Both helpers live in **`R/anchored.R`** alongside the other anchor computation
functions.

### `.somalign_orthogonal_residual(D, V)`

```r
# D: n_anchors × p matrix (scaled displacements)
# V: p × rank matrix (orthonormal batch subspace from .somalign_subspace_svd)
# Returns R = D (I_p - V Vᵀ), same dimensions as D.
.somalign_orthogonal_residual <- function(D, V) {
  D - D %*% V %*% t(V)
}
```

This is algebraically the projection of each row of D onto the orthogonal complement
of span(V). No new dependencies; pure base R. The form `D - D V Vᵀ` avoids
materialising the p × p identity matrix (saves memory when p is large).

### `.somalign_permutation_null(D, V, n_perm, seed)`

```r
# D: n_anchors × p matrix
# V: p × rank matrix
# n_perm: positive integer
# seed: integer or NULL
# Returns numeric vector of length n_perm (leading SV under each permutation).
.somalign_permutation_null <- function(D, V, n_perm, seed) {
  if (!is.null(seed)) set.seed(seed)
  n <- nrow(D)
  sv_null <- numeric(n_perm)
  for (b in seq_len(n_perm)) {
    D_perm <- D[sample.int(n), , drop = FALSE]   # row-shuffle of D
    R_perm <- D_perm - D_perm %*% V %*% t(V)
    sv_null[b] <- svd(R_perm, nu = 0L, nv = 0L)$d[1L]
  }
  sv_null
}
```

## 5. Algorithm

### Why row-shuffle of D (not sign-flip of R)

The permutation breaks the within-anchor-pair correspondence by shuffling
the rows of `anchor_old_scaled` while keeping `anchor_new_scaled` fixed —
equivalently, it shuffles the rows of D = anchor_old_scaled − anchor_new_scaled.
This is the natural exchangeability unit: under H₀ the pairing between an
old-batch anchor and its new-batch counterpart carries no information beyond what
either marginal distribution carries. Row-shuffling respects the marginal
distribution of `anchor_old_scaled` rows exactly (no parametric assumption) and
is the same operation used in the original idea document (§Approach 3).

Sign-flipping of R (the residual) is *not* used because it would test a different
null — that R has no row-level structure at all — whereas the goal is to test
whether the *pairing* between old and new batches introduces structure in the
orthogonal complement. The row-shuffle targets the pairing, making it the
correct exchangeability operation.

### Test statistic

The statistic is the leading singular value of R = D (I − V Vᵀ).

**Why leading SV rather than ||R||_F relative to ||D||_F?**
The leading SV is sensitive to *structured* signal (a low-rank biology direction
in the residual) without being inflated by isotropic noise across many dimensions.
The Frobenius-ratio alternative would be large whenever p is large and D has any
residual variance at all, making it poorly calibrated for high-dimensional CyTOF
data. The leading SV captures the specific concern: is there one dominant
biology direction that the subspace missed?

### Complete R snippet for the test body

```r
somalign_exclusion_test <- function(fit, n_perm = 999L, seed = 1L) {
  ## --- input checks ---
  if (!inherits(fit, "somalign_anchored_fit"))
    stop("`fit` must be a somalign_anchored_fit object.", call. = FALSE)
  if (!fit$anchors$correction %in% c("subspace", "both"))
    stop("`somalign_exclusion_test` requires correction = 'subspace' or 'both'.",
         call. = FALSE)
  bs <- fit$anchors$batch_subspace
  if (is.null(bs))
    stop("No batch subspace found in `fit$anchors$batch_subspace`.", call. = FALSE)
  D <- fit$anchors$displacements
  if (is.null(D))
    stop("`fit$anchors$displacements` is NULL. Refit with a version of somalign ",
         ">= 0.99.2 that stores displacement matrices.", call. = FALSE)
  .somalign_check_pos_int(n_perm, "n_perm")
  if (!is.null(seed)) .somalign_check_pos_int(seed, "seed")

  V <- bs$V                      # p × rank
  n <- nrow(D)
  p <- ncol(D)
  r <- bs$rank

  ## --- power warning ---
  if (n < r + 2L)
    warning(sprintf(
      "n_anchors (%d) < rank + 2 (%d): test has essentially no power. ",
      n, r + 2L), "Recommend >= 3 * rank anchor pairs.", call. = FALSE)

  ## --- observed statistic ---
  R_obs <- .somalign_orthogonal_residual(D, V)
  sv_obs <- svd(R_obs, nu = 0L, nv = 0L)$d[1L]

  ## --- permutation null ---
  sv_null <- .somalign_permutation_null(D, V, n_perm, seed)

  ## --- summaries ---
  p_val <- mean(sv_null >= sv_obs)
  nq    <- quantile(sv_null, probs = c(0.025, 0.5, 0.975), names = TRUE)
  rel   <- if (nq["50%"] > 0) sv_obs / nq["50%"] else NA_real_
  feat_norm <- sqrt(colSums(R_obs^2))
  names(feat_norm) <- colnames(D)

  verdict <- if (p_val > 0.10) "pass" else if (p_val >= 0.05) "warn" else "fail"

  structure(
    list(sv_observed = sv_obs, sv_null = sv_null, p_value = p_val,
         null_quantiles = nq, relative_stat = rel,
         rank_used = r, n_anchors = n, n_features = p,
         verdict = verdict, feature_residual_norm = feat_norm),
    class = "somalign_exclusion_test"
  )
}
```

## 6. Integration Points

- **Read-only consumption:** `somalign_exclusion_test()` reads `fit$anchors$batch_subspace$V`,
  `fit$anchors$displacements`, `fit$anchors$correction`, and `fit$anchors$n_anchors`.
  It does not modify `fit` in place.

- **Reuses validators:** `.somalign_check_pos_int()` (utils.R line 37) for `n_perm`
  and `seed`. No new dependencies beyond base R `svd()`, `quantile()`, `sample.int()`.

- **Validation already done:** `.somalign_validate_anchors()` (anchored.R line 252)
  has already checked that `anchor_old` and `anchor_new` are finite numeric matrices
  with matched features. Since D is computed from those validated matrices, no
  additional finite-check is needed on D.

- **Precedent for structured result objects:** `somalign_diagnostics()` returns a
  named list printed by an S3 method. The same pattern is used here.

- **Optional inline reporting:** When `fit$anchors$correction %in% c("subspace", "both")`
  and `n_anchors >= 10`, `print.somalign_anchored_fit()` (print.R) may be extended
  to note: "Run somalign_exclusion_test(fit) to validate anchor exclusion restriction."
  This is optional and cosmetic; it does not require any changes to the computation.

## 7. Edge Cases

| Case | Behaviour |
|------|-----------|
| `correction = "cost_bonus"` fit | Error: "requires correction = 'subspace' or 'both'" |
| Unanchored `somalign_fit` | Error: "must be a somalign_anchored_fit object" |
| `fit$anchors$displacements` is NULL (old fit object) | Error with message to refit |
| `n_anchors < rank + 2` | Warning about low power; test still runs |
| `n_anchors == rank` (exactly full-rank D, trivially zero residual) | R_obs is zero matrix; sv_obs = 0; sv_null likely > 0; p_value = 1.0; verdict = "pass" — mathematically correct (no evidence of orthogonal structure when D spans its own column space) |
| Degenerate D (all-zero rows, e.g., identical anchor pairs) | `.somalign_subspace_svd()` returns a rank-1 zero V (utils.R line 93–94); R_obs = D; test degrades gracefully |
| `n_perm = 1` | Technically valid; no warning needed (caller's responsibility) |
| D has NA/Inf | Prevented upstream by `.somalign_validate_finite()` in `.somalign_validate_anchors()` |
| `seed = NULL` | No seeding; results non-reproducible; no error |

**Full-rank V special case in more detail:** If `rank == p` (variance_threshold = 1.0
and all singular values nonzero), then V is p × p orthonormal (i.e., V Vᵀ = I_p),
so R = D − D = 0. The leading SV is 0. All permutation nulls will also be ≈ 0
(floating-point noise). The p-value is well-defined but uninformative; add a note
in the documentation that the test has no power when V spans the full feature space.

### Test 2 — anchors with injected orthogonal biology → test flags (p small)

```r
test_that("exclusion test fails when anchors carry biology-direction displacement", {
  skip_if_not_installed("kohonen")
  fx <- make_subspace_fixture(seed = 42L)
  # Inject a large biology-direction component (c = c(0,1,0)) into anchor displacement
  cc <- fx$cc        # c(0, 1, 0): orthogonal to batch direction b = c(1,0,0)
  bio_contam <- 3.0  # large signal in orthogonal direction
  anc_new_contaminated <- fx$anc_new +
    matrix(bio_contam * cc, nrow(fx$anc_new), length(cc), byrow = TRUE)
  fit_contaminated <- somalign_fit_anchored(
    fx$qry, fx$ref,
    anchor_old = fx$anc_old, anchor_new = anc_new_contaminated,
    rho_anchor = 1, correction = "subspace"
  )
  result <- somalign_exclusion_test(fit_contaminated, n_perm = 499L, seed = 1L)
  # Large orthogonal component → small p-value
  expect_lt(result$p_value, 0.05)
  expect_equal(result$verdict, "fail")
  # cc-direction (F2) should dominate the feature residual norm
  expect_equal(which.max(result$feature_residual_norm), 2L)
})
```

| Roxygen docs + NAMESPACE | ~30 lines | 30 min |
| Tests (4 test cases) | ~80 lines | 60 min |
| **Total** | ~170 lines | ~3.5 hours |

### Risks

1. **Low-power regime:** With n_anchors < 3 × rank the permutation null has few
   distinct values and the p-value quantisation is coarse. The power warning (§5)
   mitigates this; the function still runs and returns a valid (if imprecise) p-value.

2. **D-storage dependency on Idea 7:** Both ideas need `fit$anchors$displacements`.
   The edit in §3 is the same two-line change. If Idea 7 is implemented first, this
   plan inherits D-storage for free. If implemented first, Idea 7 should reuse the
   same slot name. The two plans should be coordinated so the edit is not duplicated.

3. **V is fixed at test time:** The permutation null shuffles rows of D but
   recomputes R_perm using the *original* V (not a V recomputed from D_perm).
   This is intentional: V was estimated from the real data and represents the
   batch direction. Recomputing V from D_perm would introduce additional variance
   into the null and make the test conservative. The fixed-V approach is standard
   for permutation tests of projection residuals.

4. **`seed` management:** `set.seed()` inside the helper function changes global
   RNG state. This is consistent with other somalign functions that accept a `seed`
   argument. Document this behaviour in the roxygen `@details`.

5. **Permutation null is one-sided by construction:** The p-value is
   `mean(sv_null >= sv_obs)`. A two-sided test is not meaningful here because a
   *smaller* observed SV than the null would simply mean the residual has less
   structure than noise, which is not a concern.

### Dependencies

- No new R package dependencies. Uses base R: `svd()`, `sample.int()`, `quantile()`,
  `set.seed()`.
- No changes to any existing computational path (OT, Sinkhorn, node shifts,
  results projection).
- Backwards-compatible: existing `somalign_anchored_fit` objects from prior package
  versions will lack `fit$anchors$displacements`; the function detects this and
  errors with a clear message to refit.
